クラウドGPUの比較について

序論

  • 機械学習：多量のデータサンプル（トレーニングデータ）を使って、モデル（人工ニューラルネットワークなど）を学習させる。
  • トレーニングデータの量が多いため、学習は計算量が大きく、計算は時間がかかる。
    例： Imagenet 、 GTX580 x2 GPUs　（~3000GFLOPS）　 −　5-6日。
  • CPUよりGPUの方が（単精度）性能が数倍高い、GPUで行う学習は数倍早く終わる。
  • GPUのモデルにより、値段も性能も違う。市販GPUの例：
    GPU モデル 性能 価格
    MCI GT 710 (NVIDIA/GeForce GT 710) 366 GFLOPS ~3000円
    NVIDIA Quadro GP100 10600 GFLOPS ~925000円
    CPUの性能: ~100 - 1200 GFLOPS
    List of Nvidia graphics processing units     グラフィックボード・ビデオカード スペック検索
  • 機械学習用の計算サーバーは高価　＝＞　クラウドを使う必要がある。

クラウドGPU比較：プロジェクトの目的

  • クラウドでGPUが載っているマシーンの選択肢が多く、性能とコストの見積もりは困難。
  • コストの比較：分ごとに課金されるプランもあれば、年ごとに課金されるプランもあり、初期費用が必要なプランも、使用できるノード時間が制限されるプランもある。特定の期間にマシーンを借りたい場合、コストだけで選ぶ場合でも、一番安いプランを選ぶのは難しい。
  • 性能の高いマシーンは、コストもより高いけど、計算はより早く終わるので、性能の低めのマシーンよりは結局やすくなるのでは？
  • 複数のクラウドGPUマシーンのコストと性能を容易に比較できるためのグラフを作成するのはプロジェクトの目的。

実現

  • 複数のプロバイダーのウエブサイトなどからGPUオファーについての情報を収集した。
  • 収集した情報を元に以下のインタラクティブグラフを作成した。
    1. クラウドマシーンのCPU性能、GPU性能とメモリ量を比較するグラフ。
    2. 各オファーの利用期間・コストグラフ。
    3. コストグラフで選択した期間の断面グラフ：各オファーのコスト比較グラフ。
    4. 特定の利用期間のための各オファーのCPUとGPUの1TFLOPSあたりのコストのグラフ。
    5. 特定の計算量とマシーンの数にかかる、各オファーのおよその計算時間とコストのグラフ。
  • 上記のグラフの目的は以下の通り。
    1. 各マシーンの性能を推定し比較できる。
    2. 利用期間によって、各オファーの変動するコストを比較できる。利用期間の目安があれば一番安いオファーがわかる。
    3. 同上
    4. CPUとGPU性能を考慮した、各オファーのコストパフォーマンスの比較ができる。
    5. 計算量が増えることに連れて、計算時間が増え、コストも上がる。性能の高いマシーン、または複数のマシーンを借りれば、計算は早く終わる。多少コストを犠牲にし、早く計算を終わらせたい場合もある。計算にかかる時間とコストの目安がわかれば、特定の計算課題に一番あっているマシーンとその数を選択できる。
  • 比較するオファーをプロバイダー、メモリ量、GPUモデルなどで制限するためのフィルタリング用のグラフを作成した。

今後の課題

  • 特定の計算課題の計算量を測定し、自動的に一番安いオファーを選択する仕組みの実現。
  • 上記の機能に加えて、自動的にクラウドで計算を行う仕組みの実現。
